Briker DDE

Briker DDE merupakan aplikasi yang menghubungkan antara sistem telepon asterisk (briker) dengan service DDE (Dynamic Data Exchange). Pada intinya aplikasi ini akan menampilkan popup jika ada telepon yang masuk ke extension/agent. Popup yang ditampilkan bisa dikonfigurasi sesuai keinginan data apa saja yang akan ditampilkan sebagai popup. Entah itu nama penelpon, alamat, nomor pelanggan dll. Berikut ini adalah daftar fitur beserta penjelasan dari aplikasi Briker DDE.


1. Menu Service

Menu service ini digunakan untuk menjalankan atau memberhentikan fitur pooling data ke briker. Jika ada telepon masuk ke extension masing-masing agent, Maka aplikasi ini akan menampilkan popup data penelpon. Dan juga dimenu ini ditampilkan log aplikasi. Di log tersebut tercatat jalanya proses pooling data ke briker.


2. Menu Konfigurasi Server

Menu server ini adalah menu untuk setting konfigurasi ke sistem asterisk/briker. Didalam menu tersebut yang harus diisi adalah alamat server API briker, username/password serta extension dari agent yang menggunakan aplikasi ini.


3. Menu Konfigurasi Datasource

Menu konfigurasi datasource ini digunakan untuk koneksi ke database. Tujuanya adalah untuk mencari data member/customer berdasarkan caller_id dari orang/member/customer yang menelpon ke agent. Jika member/customer di database tersebut ditemukan, maka data customer tersebut akan ditampilkan sebagai popup notifikasi. Data yang ditampilkan sesuai dengan query yang di setting.


4. Menu Konfigurasi DDE Service Incoming/Outgoing

Menu konfigurasi DDE ini merupakan menu yang digunakan untuk komunikasi antara aplikasi briker-dde dengan Service DDE pada windows. Service DDE ini bisa di aktifkan bisa juga tidak, tergantung keinginan. Jika Activate DDE di centang, maka ketika ada telepon masuk ke agent, aplikasi briker-dde ini akan mengirim command ke DDE Service Name yang di simpan dalam konfigurasi. Jika Activate Outgoing DDE di centang, maka aplikasi briker-dde akan menjalankan listener untuk menerima command yang dikirim dari Service DDE untuk diteruskan ke briker. Agar briker menjalankan command click-to-call ke nomor yang dikirimkan oleh Service DDE.


5. License

Jika ingin menggunakan aplikasi briker-dde ini harus mendaftarkan ID terlebih dahulu ke administrator/vendor. Karena aplikasi ini menggunakan sistem license agar bisa dipakai oleh user.
